# 📘 COMPLAINT MANAGEMENT APP – FEATURE DESCRIPTION

---

# 🔐 1. Login System (Database-Based Authentication)

**Description:**
The login system allows users to enter a username and password. Instead of hardcoding credentials, the app checks the entered data against records stored in the SQLite database.

**How it works:**

* User enters credentials
* App queries database (`users` table)
* If match found → login successful → Dashboard
* Else → error message shown

**Implemented in:**

* `LoginActivity.java` → login logic (`checkUser()` call)
* `DBHelper.java` → `checkUser()` method
* `activity_login.xml` → UI

---

# 📝 2. Signup / Registration System

**Description:**
New users can create an account by entering a username and password. This data is stored in the database and later used for login.

**How it works:**

* User enters new credentials
* Data inserted into `users` table
* User can now log in using these credentials

**Implemented in:**

* `SignupActivity.java` → handles input + insert
* `DBHelper.java` → `insertUser()`
* `activity_signup.xml` → UI
* `AndroidManifest.xml` → activity declaration

---

# 🗄️ 3. Database (SQLite Storage)

**Description:**
SQLite is used to store all persistent data in the app.

**Tables used:**

* `users` → stores login credentials
* `complaints` → stores complaint details

**Operations performed:**

* Insert (new complaint/user)
* Fetch (pending complaints)
* Check (login validation)

**Implemented in:**

* `DBHelper.java`

  * `onCreate()` → table creation
  * `insertComplaint()`
  * `insertUser()`
  * `getPending()`
  * `checkUser()`

---

# 📱 4. Dashboard Screen (Navigation Hub)

**Description:**
After login, the dashboard provides navigation to different parts of the app using buttons.

**Options available:**

* Register Complaint
* Add Complaint Details
* View Pending Complaints

**Implemented in:**

* `DashboardActivity.java` → button click navigation
* `activity_dashboard.xml` → UI

---

# 🧾 5. Register Complaint Screen

**Description:**
This screen allows users to register a complaint with details like date, type, and status.

**Features used:**

* DatePicker → select date
* Spinner → choose complaint type
* Checkbox → mark resolved/unresolved

**How it works:**

* User fills details
* Data is inserted into `complaints` table

**Implemented in:**

* `RegisterActivity.java`
* `activity_register.xml`
* `DBHelper.java` → `insertComplaint()`

---

# ⚠️ 6. Complaint Input Screen

**Description:**
This screen collects complaint-related numeric details needed for risk calculation.

**Inputs:**

* Complaint ID
* Complaint date
* Number of people affected

**How it works:**

* Takes input
* Sends number of people to next screen using Intent

**Implemented in:**

* `ComplaintActivity.java`
* `activity_complaint.xml`

---

# 🧮 7. Risk Calculation Feature

**Description:**
Calculates risk level based on number of people affected.

**Formula used:**

> Risk = (Number of People × 5) + 10

**How it works:**

* Receives data via Intent
* Applies formula
* Displays result

**Implemented in:**

* `RiskActivity.java`
* `activity_risk.xml`

---

# 📋 8. Pending Complaints Display

**Description:**
Displays all complaints that are not yet resolved.

**How it works:**

* Fetches records where `resolved = 0`
* Displays them using ListView

**Implemented in:**

* `PendingActivity.java`
* `activity_pending.xml`
* `DBHelper.java` → `getPending()`

---

# 🔁 9. Navigation Using Intents

**Description:**
The app moves between screens using Android Intents.

**Flow:**

* Login → Dashboard
* Dashboard → Other screens
* Complaint → Risk screen

**Implemented in:**

* All activity files using `startActivity()`

---

# 🔔 10. Toast Messages (User Feedback)

**Description:**
Toast messages provide quick feedback to the user for actions like login success, errors, and data saving.

**Examples:**

* “Login Successful”
* “Invalid Credentials”
* “Complaint Saved”

**Implemented in:**

* All activity files (`LoginActivity`, `RegisterActivity`, etc.)

---

# 🔄 11. Logout / Go to Login Feature

**Description:**
Allows user to return to login screen and clears previous screens from memory.

**How it works:**

* Button click → opens LoginActivity
* Clears back stack using `finishAffinity()`

**Implemented in:**

* `PendingActivity.java`
* `activity_pending.xml`

---

# 🎯 FINAL OVERVIEW

This app demonstrates:

* User authentication using SQLite
* Data storage and retrieval
* Multiple screens with navigation
* UI components (Spinner, DatePicker, ListView)
* Basic business logic (risk calculation)
